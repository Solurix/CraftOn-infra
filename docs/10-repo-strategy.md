# 10 — Repository Strategy

CRAFT-ON is a **single monorepo** (`Solurix/CraftOn-infra`), per
[`adr/0010-monorepo.md`](adr/0010-monorepo.md), which superseded the original
multi-repo decision (`adr/0003-multi-repo.md`).

## Layout

```
CraftOn-infra/            the monorepo
├── api/                  FastAPI backend — SQLAlchemy + Alembic, pytest
├── web/                  Next.js PWA — TypeScript, React (workers + contractors + admin)
├── infra/terraform/      GCP IaC (environments/{dev,prod} + modules/)
├── docs/                 product / architecture / governance — source of truth
├── scripts/              check.sh (the quality gate) + api.sh helper
├── .github/              consolidated workflows (ci, preview-deploy, preview-cleanup) + CODEOWNERS
├── Makefile              single containerized entrypoint (make up / check / tf-* / gcp-*)
├── docker-compose.yml    local stack: db + api + web
└── .env.example          root env template for the local stack
```

> Admin UI lives inside `web/` (a gated `/admin` area), not a separate app.
> There is **no mobile app** — the installable PWA is the Android/iOS story
> (the former `crafton-mobile` repo was dropped; see ADR 0010).

## What goes where

- **DB schema & migrations:** `api/migrations/` (Alembic). The narrative model is in
  `docs/05-data-model.md` — keep them in sync **in the same change**.
- **API contract:** authoritative OpenAPI is generated by the FastAPI app in `api/`;
  the human overview is `docs/06-api-contract.md`. The web client's typed schema
  (`web/src/lib/api/schema.ts`) is generated from it (`npm run gen:api`) — never
  hand-edited.
- **Infra/Terraform:** `infra/terraform/`.
- **Config/flag definitions:** documented in `docs/07-config-and-flags.md`; implemented
  in `api/` (read through `ConfigService`, never hardcoded).
- **Decisions:** `docs/adr/`.

## Conventions

- Default branch: `main`. Work on a `claude/<slug>` branch, open a PR, get the single
  `ci` check green, verify on the preview, and let the owner review + merge (no
  auto-merge). See `docs/12-preview-environments.md` and the root `CLAUDE.md`.
- One CI pipeline (`.github/workflows/ci.yml`, path-filtered: `api` / `web` / `smoke`
  jobs, aggregating `ci` gate). One preview pipeline deploys both services per PR.
- Conventional, descriptive commit messages.
- Secrets via GCP Secret Manager; `.env.example` committed, real `.env` gitignored.

## History

The project previously used four repos (`crafton`, `crafton-api`, `crafton-web`,
`crafton-mobile`). They were consolidated into this monorepo in 2026-07 (ADR 0010) by
clean copy; the old repos remain as read-only archives with a deprecation notice
pointing here. Their git history stays in those repos.
